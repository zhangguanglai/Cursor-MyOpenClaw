## 🚀 下一步计划（3 个月路线图）

### 第一阶段：OpenClaw Studio MVP（2-3 周）

#### 1.1 实现基础 CLI 接口
**目标**：提供命令行工具，跑通第一个完整闭环

**任务清单**：
- [ ] 创建 CLI 入口 (`openclaw_cli.py`)
- [ ] 实现需求管理命令：
  - `openclaw create-case <title>` - 创建新需求
  - `openclaw list-cases` - 列出所有需求
  - `openclaw show-case <id>` - 查看需求详情
- [ ] 实现规划命令：
  - `openclaw plan <case-id>` - 生成实现计划
  - `openclaw show-plan <case-id>` - 查看计划
- [ ] 实现编码命令：
  - `openclaw code <case-id> <task-id>` - 生成代码补丁
  - `openclaw show-patches <case-id>` - 查看所有补丁
- [ ] 实现测试命令：
  - `openclaw test <case-id>` - 生成测试建议
  - `openclaw verify <case-id>` - 查看验收清单

**技术栈**：
- Python + `click` 或 `argparse` 构建 CLI
- SQLite 存储案例元数据
- 文件系统存储计划/补丁/测试文档

#### 1.2 实现案例库结构
**目标**：建立标准化的案例存储和组织方式

**任务清单**：
- [ ] 创建 `cases/` 目录结构
- [ ] 实现案例数据库（SQLite）：
  - `cases` 表：需求基本信息
  - `plans` 表：计划关联
  - `tasks` 表：子任务列表
  - `agent_runs` 表：Agent 调用记录
  - `test_records` 表：测试记录
- [ ] 实现文件存储结构：
  ```
  cases/
    <case-id>/
      plan.md          # 实现计划
      plan.json        # 结构化任务列表
      patches/         # 代码补丁
        task-1.patch
        task-2.patch
      tests/           # 测试建议
        suggestions.md
        checklist.md
      summary.md       # 案例总结
  ```

#### 1.3 跑通第一个完整闭环
**目标**：使用 OpenClaw 完成一个真实需求的开发

**任务清单**：
- [ ] 选择一个真实但简单的需求（例如：添加日志功能）
- [ ] 使用 CLI 创建案例
- [ ] 调用 PlanningAgent 生成计划
- [ ] 人工 Review 并确认计划
- [ ] 按子任务调用 CodingAgent 生成补丁
- [ ] 应用补丁并测试
- [ ] 调用 TestAgent 生成测试建议
- [ ] 完成验收并记录总结
- [ ] 沉淀为第一个案例模板

## 🔄 迭代原则

1. **小步快跑**：每个阶段先实现 MVP，再逐步完善
2. **实际使用**：每个功能都要在实际场景中使用和验证
3. **持续改进**：根据使用反馈快速迭代
4. **文档先行**：重要功能先写文档，再实现代码
5. **测试保障**：新功能都要有测试覆盖

---

## 💡 建议

1. **先跑通闭环，再优化体验**：第一个闭环可能很粗糙，但一定要完整
2. **优先 CLI，再 Web**：CLI 更容易快速迭代，Web 可以基于 CLI 的经验
3. **真实需求驱动**：用真实需求来验证和优化系统
4. **记录一切**：每个决策、每个问题都要记录，形成知识库
