# 执行视图功能测试指南

## 🧪 测试准备

### 1. 启动服务

```bash
# 终端 1: 启动后端服务
python start_backend.py

# 终端 2: 启动前端服务
cd openclaw-studio-frontend
npm run dev
```

### 2. 访问应用

- 前端地址: http://localhost:5173
- 后端 API 文档: http://localhost:8000/docs
- 健康检查: http://localhost:8000/health

## ✅ 测试用例

### 测试 1: 访问执行视图

1. **步骤**:
   - 访问 http://localhost:5173/cases
   - 选择一个案例（或创建新案例）
   - 点击"执行"或访问 `/cases/{caseId}/execution`

2. **预期结果**:
   - ✅ 页面正常加载
   - ✅ 显示"生成补丁"卡片
   - ✅ 显示"补丁列表"卡片
   - ✅ 显示"补丁详情"卡片

### 测试 2: 任务列表加载

1. **步骤**:
   - 进入执行视图
   - 查看"生成补丁"区域的任务选择下拉框

2. **预期结果**:
   - ✅ 任务列表正常加载（如果案例有计划）
   - ✅ 显示加载状态（如果正在加载）
   - ✅ 显示任务标题和 ID

### 测试 3: 生成补丁

1. **前置条件**:
   - 案例已有计划
   - 计划中有待处理的任务

2. **步骤**:
   - 在"生成补丁"区域选择任务
   - 点击"生成补丁"按钮

3. **预期结果**:
   - ✅ 按钮显示加载状态
   - ✅ 生成成功后显示成功消息
   - ✅ 补丁列表自动刷新
   - ✅ 新生成的补丁出现在列表中

### 测试 4: 补丁列表筛选

1. **前置条件**:
   - 案例已有补丁

2. **步骤**:
   - 在"补丁列表"控制栏中：
     - 输入文件路径进行筛选
     - 选择任务进行筛选
     - 选择排序字段和顺序

3. **预期结果**:
   - ✅ 筛选功能正常工作
   - ✅ 排序功能正常工作
   - ✅ 列表实时更新

### 测试 5: Diff 预览

1. **前置条件**:
   - 案例已有补丁

2. **步骤**:

   - ✅ Diff 内容正确显示

### 测试 6: 复制补丁功能

1. **前置条件**:

2. **步骤**:

## 🐛 已知问题和注意事项

### 1. react-diff-view 使用

- `react-diff-view` 的 API 可能与文档示例略有不同
- 如果遇到解析错误，检查补丁格式是否正确
- 确保补丁内容是标准的 diff 格式

### 2. 补丁 ID

- 补丁 ID 默认使用 task_id
- 如果 task_id 不存在，会生成临时 ID
- 这可能导致补丁应用功能在某些情况下不可用

### 3. 补丁应用

- 当前只是标记为已应用状态
- 实际应用逻辑需要后续实现
- 应用状态会保存在前端状态中，刷新页面后会丢失（需要后端支持）

## 🔧 故障排除

### 问题 1: 任务列表为空

**可能原因**:
- 案例没有计划
- 计划中没有任务

**解决方法**:
- 先访问规划视图生成计划
- 确保计划中有任务

### 问题 2: Diff 预览不显示

**可能原因**:
- 补丁格式不正确
- react-diff-view 解析失败

**解决方法**:
- 检查补丁内容格式
- 查看浏览器控制台错误信息
- 尝试使用简单的补丁内容测试

### 问题 3: 补丁应用失败

**可能原因**:
- 补丁 ID 不存在
- 后端 API 未实现

**解决方法**:
- 检查补丁是否有有效的 ID
- 检查后端 API 是否正常
- 查看网络请求错误信息
