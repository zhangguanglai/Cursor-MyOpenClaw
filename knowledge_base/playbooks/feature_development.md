# 功能开发工作流

## 概述

本文档描述使用 OpenClaw 开发新功能的完整流程。

## 工作流程

### 1. 需求创建

在 OpenClaw Studio 中创建新案例：

- 填写标题和描述
- 关联 Git 仓库和分支
- 设置验收标准

### 2. 生成计划

使用 PlanningAgent 生成实现计划：

- 分析需求
- 拆解任务
- 评估风险
- 生成计划文档

### 3. Review 计划

人工审查计划：

- 检查任务拆解是否合理
- 评估风险等级
- 调整计划内容

### 4. 执行任务

对每个任务：

1. 选择任务
2. 使用 CodingAgent 生成补丁
3. Review 补丁内容
4. 应用补丁到代码库
5. 运行测试

### 5. 测试和验收

使用 TestAgent 生成测试建议：

- 查看潜在问题
- 执行测试用例
- 完成验收清单

### 6. 完成和归档

- 更新案例状态为"已完成"
- 编写案例总结
- 归档到知识库

## 检查点

- [ ] 计划已 Review 并确认
- [ ] 所有任务已完成
- [ ] 测试通过
- [ ] 验收清单完成
- [ ] 案例总结已编写

## 常见问题

### Q: 计划不合理怎么办？

A: 可以手动编辑计划，或重新生成计划。

### Q: 补丁质量不好怎么办？

A: 可以重新生成补丁，或手动修改代码。

### Q: 测试失败怎么办？

A: 根据 TestAgent 的建议修复问题，然后重新测试。
