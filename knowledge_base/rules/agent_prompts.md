# Agent 提示词规范

## 通用原则

1. **清晰明确**: 提示词应该清晰明确，避免歧义
2. **结构化**: 使用结构化的格式（Markdown、JSON 等）
3. **上下文完整**: 提供足够的上下文信息
4. **示例驱动**: 提供示例帮助 Agent 理解期望输出

## PlanningAgent 提示词

### 输入格式

```
需求描述: <需求描述>
相关文件: <文件列表>
项目结构: <项目结构信息>
```

### 输出格式

- Markdown 格式的实现计划
- JSON 格式的任务列表
- 包含风险评估和测试建议

## CodingAgent 提示词

### 输入格式

```
任务描述: <任务描述>
相关文件: <文件内容>
代码约束: <约束条件>
```

### 输出格式

- Unified diff 格式的补丁
- 补丁说明
- 潜在风险提示

## TestAgent 提示词

### 输入格式

```
变更内容: <代码变更>
相关文件: <文件列表>
需求描述: <需求描述>
```

### 输出格式

- 潜在问题列表（含严重级别）
- 测试用例列表
- 验收清单

## 最佳实践

1. **迭代优化**: 根据 Agent 输出质量持续优化提示词
2. **版本控制**: 提示词模板应该纳入版本控制
3. **文档化**: 记录提示词的变更历史和效果
