# 完整工作流测试指南

## 🎯 测试目标

验证 OpenClaw Studio 的完整开发流程，从创建案例到归档的所有步骤。

## 🚀 快速测试方法

### 方法 1: 使用自动化测试脚本（推荐）

```bash
# 确保前后端服务已启动
# 然后运行测试脚本
python scripts/test_complete_workflow.py
```

脚本会自动执行：
1. ✅ 创建案例
2. ✅ 生成计划
3. ✅ 生成代码补丁（第一个任务）
4. ✅ 生成测试建议
5. ✅ 查看历史记录

### 方法 2: 手动测试（推荐用于详细体验）

按照以下步骤手动操作：

#### 步骤 1: 创建案例（1 分钟）

1. 访问 http://localhost:5173/cases
2. 点击 **"+ 创建案例"** 按钮
3. 填写案例信息：
   ```
   标题: 实现待办事项管理
   描述: 实现一个简单的待办事项管理功能，包括添加、完成、删除功能
   ```
4. 点击 **"创建"**

#### 步骤 2: 生成计划（2-3 分钟）

1. 在案例列表中点击刚创建的案例
2. 进入 **"规划视图"**（或直接访问 `/cases/{caseId}/plan`）
3. 点击右上角 **"生成计划"** 按钮
4. 等待生成完成（10-30 秒）
5. **验证点**：
   - [ ] 计划 Markdown 内容显示正常
   - [ ] 任务列表显示正常
   - [ ] 任务数量合理（通常 5-10 个）
   - [ ] 可以切换"计划"和"任务列表"标签

#### 步骤 3: 生成代码补丁（2-3 分钟）

1. 切换到 **"执行视图"**（或访问 `/cases/{caseId}/execution`）
2. **验证点**：
   - [ ] 任务下拉框显示所有任务
   - [ ] 可以选择任务
3. 选择一个任务
4. 点击 **"生成补丁"** 按钮
5. 等待生成完成（10-30 秒）
6. **验证点**：
   - [ ] 补丁列表显示新生成的补丁
   - [ ] 补丁信息显示正确（文件路径、描述等）
   - [ ] 可以点击"查看"预览代码差异
   - [ ] Diff 预览显示正常
   - [ ] 可以复制补丁内容

#### 步骤 4: 生成测试建议（2-3 分钟）

1. 切换到 **"测试视图"**（或访问 `/cases/{caseId}/test`）
2. 点击 **"生成测试建议"** 按钮
3. 等待生成完成（10-30 秒）
4. **验证点**：
   - [ ] 潜在问题列表显示
   - [ ] 可以按严重程度筛选
   - [ ] 测试用例列表显示
   - [ ] 可以展开查看测试用例详情
   - [ ] 验收清单显示
   - [ ] 可以勾选验收项
   - [ ] 进度条显示正确
   - [ ] 可以保存验收状态

#### 步骤 5: 查看历史记录（1 分钟）

1. 切换到 **"历史视图"**（或访问 `/cases/{caseId}/history`）
2. **验证点**：
   - [ ] 历史时间线显示
   - [ ] 可以看到规划、编码、测试等事件
   - [ ] 可以按类型筛选
   - [ ] 可以按时间范围筛选
   - [ ] 可以搜索关键词
   - [ ] 统计信息显示正确

#### 步骤 6: 查看知识库（可选）

1. 点击左侧导航的 **"知识库"**
2. **验证点**：
   - [ ] 知识库内容显示
   - [ ] 可以浏览不同类别
   - [ ] 可以搜索内容
   - [ ] 归档的案例可以查看

## 🐛 常见问题测试

### 测试错误处理

1. **访问不存在的案例**
   - 访问 `/cases/invalid-id/plan`
   - 应该显示友好的错误提示

2. **在没有计划时生成代码**
   - 直接访问执行视图
   - 应该提示先生成计划

3. **在没有补丁时生成测试**
   - 直接访问测试视图
   - 应该提示先生成补丁

4. **网络错误**
   - 停止后端服务
   - 前端应该显示错误提示和重试按钮
