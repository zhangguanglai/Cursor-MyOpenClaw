# 重复案例问题说明

## ❓ 为什么有两个相同的案例但 ID 不同？

### 原因分析

1. **案例 ID 生成机制**
   - 系统使用 `uuid.uuid4().hex[:8]` 生成唯一 ID
   - 每次创建案例都会生成新的 UUID，即使标题和描述完全相同
   - **这是设计行为**：允许创建相同标题的案例（适用于需要创建多个相似但不同案例的场景）

2. **没有重复检测机制（已修复）**
   - 原系统设计没有检查重复
   - 如果脚本运行两次，或前端和后端同时创建，就会产生重复案例

3. **您的具体情况**
   - `case-2a73c7be` - 创建时间: 2026-03-06 08:43:21
   - `case-e13da6ed` - 创建时间: 2026-03-06 08:43:31
   - 时间差约 10 秒，可能是脚本被运行了两次

## ✅ 已实施的解决方案

### 1. 添加重复检测机制

**位置**: `openclaw_studio/case_manager.py`

现在创建案例时会：
- 检查是否已存在相同标题的案例（非归档状态）
- 如果存在，返回现有案例而不是创建新的
- 避免创建重复案例

### 2. 添加案例删除功能

**API 端点**: `DELETE /api/v1/cases/{case_id}`

可以删除不需要的重复案例。

**注意**: 删除操作会同时删除：
- 案例记录
- 计划、任务、补丁
- 测试记录、Agent 运行记录
- 文件存储目录

## 🔧 如何删除重复案例

### 方法 1: 使用脚本（推荐）

```bash
# 删除较早创建的重复案例
python scripts/delete_case.py case-2a73c7be
```

### 方法 2: 使用 API

```bash
curl -X DELETE http://localhost:8000/api/v1/cases/case-2a73c7be
```

### 方法 3: 通过前端（如果已实现）

在需求中心页面，点击案例的"更多"按钮，选择"删除"。

## 📝 建议

1. **保留较新的案例** (`case-e13da6ed`)
   - 这是刚刚通过脚本创建的
   - 删除较早的案例 (`case-2a73c7be`)

2. **未来创建案例时**
   - 系统会自动检测重复
   - 如果存在相同标题的案例，会返回现有案例
   - 不会再创建重复案例

3. **如果需要强制创建**
   - 可以修改标题使其不同
   - 或者设置 `check_duplicate=False`（不推荐）

## ⚠️ 注意事项

- **删除操作不可逆**：删除案例会同时删除所有关联数据
- **需要重启后端服务**：新的删除 API 需要重启后端才能生效
- **重复检测规则**：只检查标题，不检查描述内容
