# OpenClaw Studio 用户使用指南

## 🚀 快速开始

### 1. 启动服务

#### 启动后端服务
```bash
# 在项目根目录执行
python start_backend.py
```

后端服务将在 `http://localhost:8000` 启动

#### 启动前端服务
```bash
# 在 openclaw-studio-frontend 目录执行
cd openclaw-studio-frontend
npm run dev
```

前端服务将在 `http://localhost:5173` 启动

### 2. 访问系统

打开浏览器访问：**http://localhost:5173**

## 📋 完整开发流程体验

### 第一步：创建案例

1. **进入需求中心**
   - 访问 http://localhost:5173/cases
   - 点击左上角的 **"+ 创建案例"** 按钮

2. **填写案例信息**
   - **标题**：输入案例标题（例如："实现用户认证系统"）
   - **描述**：详细描述需求（包括功能点、技术要求、验收标准等）
   - **仓库路径**（可选）：如果关联 Git 仓库，填写路径
   - **分支**（可选）：指定 Git 分支

3. **创建案例**
   - 点击"创建"按钮
   - 系统会自动生成案例 ID
   - 案例状态为 `created`

### 第二步：生成实现计划

1. **进入规划视图**
   - 在需求中心列表中找到刚创建的案例
   - 点击 **"查看详情"** 或直接访问规划视图
   - 或点击案例标题进入详情页，然后切换到"规划"标签

2. **生成计划**
   - 点击右上角的 **"生成计划"** 按钮
   - 系统会调用 PlanningAgent 生成实现计划
   - 等待生成完成（可能需要 10-30 秒）

3. **查看计划**
   - **计划标签**：查看完整的实现计划 Markdown
   - **任务列表标签**：查看分解的任务列表
   - 可以编辑计划内容（点击编辑按钮）

4. **管理任务**
   - 查看任务详情（点击任务）
   - 更新任务状态（pending/completed）
   - 查看任务关联的文件

### 第三步：生成代码补丁

1. **进入执行视图**
   - 在案例详情页切换到"执行"标签
   - 或直接访问：`http://localhost:5173/cases/{caseId}/execution`

2. **选择任务**
   - 在"生成补丁"区域，从下拉框选择一个任务
   - 点击 **"生成补丁"** 按钮

3. **查看补丁**
   - 系统会生成代码补丁
   - 在补丁列表中查看生成的补丁
   - 点击 **"查看"** 按钮预览代码差异（Diff）

4. **应用补丁**
   - 点击 **"应用补丁"** 按钮
   - 确认应用操作
   - 补丁状态会更新为"已应用"

5. **筛选和排序**
   - 按文件路径筛选补丁
   - 按任务 ID 筛选补丁
   - 按任务 ID 或生成时间排序

### 第四步：生成测试建议

1. **进入测试视图**
   - 在案例详情页切换到"测试"标签
   - 或直接访问：`http://localhost:5173/cases/{caseId}/test`

2. **生成测试建议**
   - 点击 **"生成测试建议"** 或 **"重新生成"** 按钮
   - 系统会调用 TestAgent 分析代码变更
   - 等待生成完成

3. **查看测试结果**
   - **潜在问题**：查看识别出的潜在问题（按严重程度筛选）
   - **测试用例**：查看生成的测试用例和步骤
   - **验收清单**：查看验收清单，勾选已完成项

4. **管理验收清单**
   - 勾选已完成的验收项
   - 点击 **"保存验收状态"** 保存进度
   - 查看完成进度百分比

### 第五步：查看历史记录

1. **进入历史视图**
   - 在案例详情页切换到"历史"标签
   - 或直接访问：`http://localhost:5173/cases/{caseId}/history`

2. **查看开发历史**
   - 查看时间线展示的开发历史
   - 按类型筛选（planning, coding, testing 等）
   - 按时间范围筛选
   - 搜索关键词

3. **查看统计信息**
   - 查看总事件数
   - 查看各类型事件数量
   - 查看详细的事件信息

### 第六步：归档案例

1. **完成案例**
   - 当所有任务完成，测试通过后
   - 将案例状态更新为 `completed`

2. **自动归档**
   - 系统会自动将案例归档到知识库
   - 案例的所有信息会被保存到知识库

3. **查看知识库**
   - 访问知识库：http://localhost:5173/knowledge
   - 查看归档的案例
   - 搜索和浏览知识库内容

## 🎯 UI 交互指引

### 导航结构

```
需求中心 (/cases)
  ├── 案例列表
  │   ├── 创建案例按钮
  │   ├── 案例卡片
  │   │   ├── 查看详情
  │   │   └── 更多操作
  │   └── 筛选和搜索
  │
  └── 案例详情
      ├── 规划视图 (/cases/{id}/plan)
      ├── 执行视图 (/cases/{id}/execution)
      ├── 测试视图 (/cases/{id}/test)
      └── 历史视图 (/cases/{id}/history)

知识库 (/knowledge)
  ├── 类别浏览
  ├── 搜索功能
  └── 内容查看
```

### 关键操作按钮

#### 需求中心
- **"+ 创建案例"**：创建新案例
- **"查看详情"**：查看案例详情
- **"更多"**：更多操作（编辑、删除等）

#### 规划视图
- **"生成计划"**：触发计划生成
- **"保存"**：保存编辑的计划内容
- **任务状态切换**：更新任务状态

#### 执行视图
- **"生成补丁"**：为选中的任务生成代码补丁
- **"应用补丁"**：标记补丁为已应用
- **"查看"**：预览代码差异
- **"复制"**：复制补丁内容

#### 测试视图
- **"生成测试建议"**：生成测试分析
- **"重新生成"**：重新生成测试建议
- **"刷新"**：刷新测试结果
- **"保存验收状态"**：保存验收清单进度

### 状态指示

- **已创建** (created)：案例刚创建，未开始
- **规划中** (planning)：正在生成或已生成计划
- **执行中** (executing)：正在生成代码补丁
- **测试中** (testing)：正在生成测试建议
- **已完成** (completed)：案例已完成并归档

## 📝 常见问题

### Q1: 如何知道案例的当前状态？

**A**: 在需求中心的案例列表中，每个案例都有一个状态标签：
- `已创建` (created)
- `规划中` (planning)
- `执行中` (executing)
- `测试中` (testing)
- `已完成` (completed)

### Q2: 生成计划需要多长时间？

**A**: 通常需要 10-30 秒，取决于：
- LLM API 响应时间
- 需求复杂度
- 网络状况

### Q3: 如何查看代码补丁的详细内容？

**A**: 在执行视图中：
1. 找到要查看的补丁
2. 点击 **"查看"** 按钮
3. 在弹出的 Modal 中查看代码差异
4. 可以切换 split/unified 显示模式

### Q4: 验收清单的进度保存在哪里？

**A**: 验收清单的进度保存在浏览器的 localStorage 中，按案例 ID 存储。刷新页面后会自动恢复。

### Q5: 如何删除案例？

**A**: 目前可以通过以下方式删除：
1. 使用脚本：`python scripts/delete_case_direct.py <case_id>`
2. 使用 API：`DELETE /api/v1/cases/{case_id}`

### Q6: 如何查看 Git 状态？

**A**: 在规划视图和执行视图中，会自动显示 Git 状态卡片，包括：
- 当前分支
- 是否有未提交更改
- 远程仓库信息
- 最新提交信息

## 🔧 故障排查

### 问题 1: 页面显示白屏

**解决方案**：
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页的错误信息
3. 查看 Network 标签页的 API 请求
4. 检查后端服务是否正常运行

### 问题 2: 生成计划失败

**可能原因**：
- LLM API Key 未配置
- 网络连接问题
- LLM API 服务不可用

**解决方案**：
1. 检查环境变量：`QWEN_API_KEY` 或 `MINIMAX_API_KEY`
2. 检查网络连接
3. 查看后端日志

### 问题 3: 补丁生成失败

**可能原因**：
- 任务信息不完整
- CodingAgent 调用失败

**解决方案**：
1. 确保任务有完整的标题和描述
2. 检查后端日志
3. 重试生成

### 问题 4: 测试建议生成失败

**可能原因**：
- 没有可用的补丁
- TestAgent 调用失败

**解决方案**：
1. 确保已生成补丁
2. 检查后端日志
3. 重试生成
