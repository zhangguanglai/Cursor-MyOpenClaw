# 规划API调试指南

## 🔍 问题诊断

根据后端日志分析，发现以下情况：

1. **后端服务正常运行**
   - 健康检查正常（200 OK）
   - 获取案例信息正常（200 OK）

2. **规划API请求超时**
   - POST 请求发送到 `/api/v1/cases/{case_id}/planning`
   - 请求超时（60秒），没有返回响应

3. **可能的原因**
   - LLMRouter 初始化失败（环境变量未设置）
   - LLM API 调用超时或失败
   - 请求处理过程中出现未捕获的异常

## 🔧 解决方案

### 步骤 1: 检查后端服务日志

查看运行后端服务的终端窗口，应该能看到：

**正常情况应该看到：**
```
初始化 LLMRouter...
成功初始化 Provider: qwen
LLMRouter 初始化成功，可用 Provider: ['qwen']
```

**如果看到错误：**
```
LLMRouter 初始化失败: 环境变量 QWEN_API_KEY 未设置，请先配置 API Key
```

### 步骤 2: 确保环境变量已设置

在**运行后端服务的终端窗口**中：

```powershell
# 检查环境变量
echo $env:QWEN_API_KEY

# 如果未设置，设置环境变量
$env:QWEN_API_KEY='sk-fe321dca0bf146ca99df33876ad56bbb'

# 验证
echo $env:QWEN_API_KEY
```

**重要**：环境变量必须在**运行后端服务的同一终端窗口**中设置。

### 步骤 3: 重启后端服务

如果环境变量是在服务启动后设置的，需要重启服务：

1. **停止后端服务**
   - 在运行后端的终端窗口按 `Ctrl+C`

2. **设置环境变量并重启**
   ```powershell
   $env:QWEN_API_KEY='sk-fe321dca0bf146ca99df33876ad56bbb'
   python start_backend.py
   ```

3. **验证启动日志**
   - 应该看到：`成功初始化 Provider: qwen`
   - 如果没有看到，说明环境变量仍未设置

### 步骤 4: 测试规划API

重启后端服务后，等待几秒钟让服务完全启动，然后：

1. **在前端测试**
   - 访问: http://localhost:5173/cases/case-e13da6ed/plan
   - 点击"生成计划"按钮
   - 等待 10-30 秒

2. **查看后端日志**
   - 应该看到 POST 请求日志
   - 应该看到规划生成的进度
   - 如果有错误，会显示详细错误信息

## 🐛 常见问题

### Q: 为什么请求会超时？

**A**: 可能的原因：
1. LLMRouter 初始化失败，导致请求卡住
2. LLM API 调用超时（网络问题或API问题）
3. 请求处理过程中出现未捕获的异常

**解决方案**：
- 查看后端日志，找到具体错误
- 检查环境变量是否正确设置
- 检查网络连接

### Q: 如何查看详细错误信息？

**A**: 
1. 查看运行后端服务的终端窗口
2. 查看最新的日志输出
3. 如果有错误，会显示详细的堆栈跟踪

### Q: 环境变量设置了但还是失败？

**A**: 
1. 确保在**运行后端服务的同一终端**中设置
2. 设置后需要**重启后端服务**
3. 验证启动日志中是否看到 Provider 初始化成功

## 📝 下一步

如果按照上述步骤操作后仍然失败：

1. **收集信息**
   - 后端服务的完整启动日志
   - 规划API请求时的日志
   - 错误堆栈跟踪

2. **检查配置**
   - `config/llm.yml` 文件是否正确
   - API Key 是否有效
   - 网络连接是否正常

3. **测试LLM连接**
   ```bash
   python scripts/test_llm_connection.py
   ```
